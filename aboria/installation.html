Installation

This software is tested on Ubuntu 14.04LTS with the GCC compiler (version 4.8.4), and Clang compiler (version 3.8.0). See our Travis CI page for more details.

You will need to install a C++ compiler with C++11 support, and the CMake build software prior to using Aboria, which you can do on a Debian-based OS using 

sudo apt-get install build-essential cmake

The only required dependency is the Boost library. Optional dependencies are The Visualization Toolkit and Eigen (version >= 3.3~beta1), both of which add extra functionality. To install all these dependencies in a Debian-based OS you can type 

sudo apt-get install libboost-dev libvtk5-dev libeigen3-dev

To build and run Aboria's testing suite, navigate to Aboria's root directory and type 

cmake .
make
ctest

To use Aboria in your own software, simply add the Aboria/src directory to your include path and include the Aboria.h header, e.g. 

#include <Aboria.h>

If you wish to use The Visualisation Toolkit or Eigen with Aboria then you will need to define the HAVE_VTK and HAVE_EIGEN compiler definitions, and include the relevant headers and link the relevant libraries. Please see the Aboria/CMakeLists.txt and Aboria/tests/CMakeLists.txt files for an example of how to do this using CMake.
